ACI.dev 开源平台是AI Agent的工具使用基础设施，它连接了600多种工具集成。该平台提供多租户认证、细粒度权限管理和动态工具发现功能，Agent 可以通过直接函数调用或统一的 MCP 服务器访问这些功能。ACI.dev 旨在解决构建生产级AI Agent在基础设施方面的挑战，例如大规模认证、工具发现和权限控制等问题，帮助开发者无需处理复杂的OAuth流程或API客户端。它兼容任何大型语言模型(LLM)框架和Agent架构，所有组件（包括后端、开发者门户和集成）均在 Apache 2.0 许可下开源。核心功能包括丰富的预构建集成、灵活的访问方式（MCP服务器或轻量级SDK）、内置的多租户 인증 和密钥管理、自然语言权限边界和动态工具发现等。常见用例涵盖个人助理、研究、销售和客户支持等Agent场景。项目提供了详细的文档、工具列表以及相关的 MCP 服务器、Python SDK 和 Agent 示例等资源的链接，并提供本地开发环境的配置指南。社区用户可以通过Discord、Twitter和LinkedIn等渠道参与讨论和贡献。最新的开发动态和贡献指南也提供了详细说明，项目欢迎新的集成请求或代码贡献。

## unionlabs / union

最小化信任、零知识的跨链协议，专为抗审查、极高安全性和去中心化金融应用设计

**作者:** [unionlabs](https://github.com/unionlabs) | **仓库:** [Github 链接](https://github.com/unionlabs/union) | **语言:** Rust

### 项目简介

Union 是一个以信任最小化和零知识证明为核心的跨链桥接协议，专注于抗审查性、高安全性和在去中心化金融（DeFi）中的应用。它不依赖受信第三方、预言机、多重签名或多方计算（MPC）。Union 实现了 IBC 协议，使其兼容 Cosmos 生态链，并连接 Ethereum、Berachain、Arbitrum 等 EVM 链。

该项目的核心设计理念是利用共识验证，确保协议的去中心化和安全性。所有重要的协议变更，包括其他链上合约的升级、连接配置、代币设置以及协议本身的演进，都将通过去中心化的治理方式来控制，以确保 Union 的发展方向与用户、验证者和运营商的利益保持一致。

Union 项目包含多个关键组件，使用不同的编程语言实现：

- `uniond`: Union 节点实现，使用 Go 语言和 CometBLS 库。
- `galoisd`: 零知识证明器实现，使用 Go 和 Gnark 库。
- `voyager`: 高性能的跨生态中继器，使用 Rust 语言。
- `hubble`: 集成了通用消息传递（GMP）功能的多生态链索引器，使用 Rust 语言。
- `cosmwasm`: CosmWasm 智能合约栈，使用 Rust 语言。
- `light-clients`: 用于各种生态系统的轻客户端实现，使用 Rust 语言。

项目代码库包含了用于快速启动和开发的文档和工具，包括使用 Nix 进行可重复构建和开发环境设置的指南。项目遵循 Apache-2.0 和 MIT 双重许可。

总而言之，Union 旨在构建一个安全、高效、去中心化的跨链基础设施，通过零知识证明和共识验证来解决现有跨链桥的信任问题，为 DeFi 和更广泛的跨链交互提供可靠的基础层。

## Atmosphere-NX / Atmosphere

**作者:** [Atmosphere-NX](https://github.com/Atmosphere-NX) | **仓库:** [Github 链接](https://github.com/Atmosphere-NX/Atmosphere) | **语言:** C++

### 项目简介

Atmosphere：Nintendo Switch的定制固件

Atmosphere是一个正在开发中的、针对任天堂Switch主机的定制固件项目。其核心目标是取代和修改Switch原有的系统组件，从而实现定制功能和更灵活的控制。

该项目由多个核心组件构成，各司其职：

- Fusée：作为第一阶段加载器，负责加载和验证第二阶段组件（自定义TrustZone）以及核心系统模块（Kernel/FIRM sysmodules），并在需要时进行修补。它取代了原始Package1loader/NX Bootloader的功能。对于7.0.0版本固件，会用到一个名为Sept的Payload来支持运行时密钥派生。
- Exosphère：定制的TrustZone，用于运行修改过的Secure Monitor。
- Thermosphère：提供EL2 EmuNAND支持，允许备份和使用虚拟化或重定向的NAND映像。
- Stratosphère：自定义系统模块集合，包括类似Rosalina风格的模块以扩展内核和提供新功能，以及加载器重实现风格的模块以挂钩关键系统操作。
- Troposphère：应用层面的Horizon OS补丁，用于实现各种自定义固件特性。

Atmosphere项目采用GPLv2许可协议，但特别注明任天堂公司可以根据其选择，以Zero-Clause BSD许可来使用为Atmosphere项目贡献的任何源代码。

项目的开发和维护主要由SciresM, TuxSH, hexkyz, 和 fincs负责。项目成功离不开众多贡献者的支持，包括switchbrew (libnx项目和相关文档工具), devkitPro (devkitA64工具链), ReSwitched Team (额外研究和工具), ChaN (FatFs模块), Marcus Geelnard (bcl-1.2.0库), naehrwert 和 st4rk (原始hekate项目), CTCaer (hekate分支和minerva_tc), m4xw (emuMMC项目), Riley (项目命名建议), hedgeberg (研究和硬件测试), lioncash (代码清理和改进), jaames (图形资源设计), 以及所有积极贡献代码的人员。

项目代码仓库活跃，最新的提交于2025年4月8日，包含对调试配置启动的支持修复，以及对19.0.0版本内核的更新。目前项目拥有15.9k星和1.3k分支，80个Releases版本，最新版本为Atmosphère 1.8.0 prerelease (发布于2024年10月16日)。项目的代码主体以C++为主，辅以C、Makefile、汇编和Python等语言。

### 项目简介

Deep-Live-Cam项目概要：实时换脸与一键视频深度伪造

Deep-Live-Cam 是一个开源项目，专注于提供实时人脸交换和仅使用一张图片即可进行视频深度伪造的功能。该项目强调易用性，宣称只需简单三步即可实现实时换脸效果。

核心功能方面，Deep-Live-Cam 支持多种应用场景：

- **嘴部遮罩 (Mouth Mask)**： 保留原始嘴部动作，以获得更精确的换脸效果。
- **人脸映射 (Face Mapping)**： 允许在同一视频中对多个人脸同时应用不同的源人脸进行替换。
- **个性化电影观看 (Your Movie, Your Face)**： 支持在实时观看电影时将任意人脸替换到角色身上，无需先处理视频。
- **直播表演 (Live Show)**： 可用于直播或表演中的人脸替换。
- **表情包制作 (Memes)**： 结合多人脸功能，方便快速制作病毒式传播的表情包。
- **在线互动 (Omegle)**： 可用于在在线视频聊天平台 surprising others。

项目提供了适用于 Windows 系统的预编译快速启动版本，尤其适合非技术用户或拥有独立 NVIDIA/AMD 显卡的用户。同时，项目也提供了详细的手动安装指南，支持 Windows、Linux 和 macOS (包括 Apple Silicon) 平台，并提供了针对 CUDA (Nvidia)、CoreML (Apple Silicon/Legacy) 和 DirectML (Windows) 等不同硬件平台的 GPU 加速配置说明。

项目作者强调伦理使用和责任。软件内置检查机制，会阻止处理不当内容（如裸露、暴力、敏感材料等）。作者承诺遵守法律和道德规范，并保留在法律要求下停止项目或添加水印的权利。但同时声明不对最终用户的行为负责，要求用户自行确保软件使用符合道德标准和法律要求，尤其在使用真实人物肖像时，必须获得许可并注明是深度伪造内容。

Deep-Live-Cam 是一款功能强大的实时人脸交换工具，提供了多种实现快速简便深度伪造的方式，但也需用户承担相应的伦理和法律责任。

### 项目简介

Cloudflare MCP 服务器项目概述

该项目名为 "cloudflare/mcp-server-cloudflare"，旨在为 Cloudflare 服务提供基于 Model Context Protocol (MCP) 的服务器实现。MCP 是一个新的标准化协议，用于管理大型语言模型 (LLMs) 与外部系统之间的上下文。通过这些 MCP 服务器，MCP 客户端（例如 Cursor, Claude）可以连接到用户的 Cloudflare 账户，并利用自然语言来执行任务。

这些服务器使 MCP 客户端能够读取用户的 Cloudflare 账户配置，处理信息，基于数据提供建议，甚至直接代为修改配置。这些功能涵盖了 Cloudflare 的多种服务，包括应用开发、安全和性能。项目包含了多个特定的 MCP 服务器，例如用于获取 Cloudflare 文档参考信息的文档服务器、用于构建 Workers 应用的 Workers Bindings 服务器、用于调试和获取应用日志分析的 Observability 服务器，以及提供全球互联网流量洞察和安全扫描的 Radar 服务器等。此外，还包括用于沙盒开发环境、浏览器渲染、Logpush 任务健康监测、AI Gateway 日志搜索、AutoRAG 文档列表与搜索、审计日志查询与报告生成、DNS Analytics 优化和数字体验监控等服务器。

用户可以通过支持远程 MCP 服务器的客户端直接输入服务器 URL 来访问这些功能，对于暂不支持的客户端，则需要通过 mcp-remote 工具进行配置。项目处于活跃开发阶段，欢迎社区贡献，并遵守 Apache-2.0 开源许可协议。项目还提供了贡献指南和故障排除建议，例如针对 Claude 响应中断的问题，建议用户保持查询简洁明了，并将复杂的请求拆分成更小的调用。同时提醒用户，部分高级功能可能需要付费的 Cloudflare Workers 计划。

AWS SDK for Java – Version 2 （简称 v2）是官方的适用于 Java 的 AWS 开发工具包。它是对 v1 的重写，并引入了一些重要的新功能，如对非阻塞 I/O 的一流支持以及可插拔的 HTTP 实现，以进一步定制应用程序。

使用该 SDK，Java 开发者可以轻松地与 Amazon Web Services 互动。推荐的集成方式是通过 Maven Central，可以使用 BOM（Bill of Materials）来管理子模块版本，也可以单独导入所需的特定服务模块，或者导入包含所有服务的完整 SDK（但不推荐，最好只包含所需模块）。

## juanfont / headscale

一个开源、自托管的 Tailscale 控制服务器实现

**作者:** [juanfont](https://github.com/juanfont) | **仓库:** [Github 链接](https://github.com/juanfont/headscale) | **语言:** Go

### 项目简介

Headscale：Tailscale 控制服务器的开源自托管实现

Headscale是一个开源项目，旨在提供Tailscale控制服务器的自托管替代方案。Tailscale是基于Wireguard的现代VPN，其控制服务器负责节点密钥交换、IP地址分配、用户边界隔离、机器共享以及路由广告。Headscale的目标是为自签用户和爱好者提供一个可以用于个人项目或小型开源组织的、独立的Tailscale网络（称为tailnet）控制服务器。

Headscale的设计目标明确，专注于实现单个tailnet，而非像官方Tailscale服务那样支持大型组织或多个tailnet。项目积极维护，并欢迎社区贡献。文档中提供了关于特性、客户端操作系统支持以及如何运行Headscale的信息。值得注意的是，项目不鼓励使用反向代理和容器来运行Headscale，建议查阅官方文档获取正确部署方式。

为了方便贡献者，项目提供了详细的贡献指南和开发环境设置建议，包括使用Nix来确保一致性。代码风格方面强制执行Go语言的lint和格式化规则，Protobuf代码也遵循规范。项目还提供了测试和构建的命令，并强调在提交时将生成的代码与修改的代码分开提交。

尽管Headscale与Tailscale公司没有直接关联，但其中一位活跃维护者受雇于Tailscale并被允许投入工作时间进行贡献。项目维护者共同决定项目方向，核心原则是服务于自托管用户、爱好者社区，并维护项目的可持续性。

项目在GitHub上开源，许可证为BSD-3-Clause，拥有大量关注者和贡献者，主要使用Go语言开发。

## koreader / koreader

支持 PDF、DjVu、EPUB、FB2 等多种格式的电子书阅读应用，可在 Cervantes、Kindle、Kobo、PocketBook 和 Android 设备上运行

**作者:** [koreader](https://github.com/koreader) | **仓库:** [Github 链接](https://github.com/koreader/koreader) | **语言:** Lua

### 项目简介

KOReader：专为电子墨水屏设备设计的增强型电子书阅读器

KOReader是一款功能强大的电子书阅读应用程序，主要面向电子墨水屏设备，但也支持Android和Linux系统。其核心优势在于广泛的文件格式兼容性，包括PDF、DjVu、EPUB、FB2、Mobi、DOC、RTF、HTML、CHM、TXT以及ZIP包中的部分格式。尤其值得一提的是，它内置了K2pdfopt库，可以对扫描版的PDF和DjVu文档进行重新排版优化，提升阅读体验。

该应用提供了高度可定制化的阅读界面和丰富的排版选项，允许用户调整页边距、行间距，并支持使用外部字体和样式。内置的多语言断字字典进一步优化了文本显示效果。KOReader还具备强大的集成能力，可以连接Calibre进行元数据搜索、无线接收电子书和浏览OPDS书库，并支持Wallabag、维基百科和谷歌翻译等在线内容服务。

KOReader针对电子墨水屏设备的特性进行了优化，界面简洁流畅，无多余动画，具备可调节的文本对比度，并支持在分页媒体中轻松缩放以适应内容或页面。其快速的翻页速度在某些旧设备上甚至优于原生阅读软件。

作为一个可扩展的平台，KOReader支持插件，进一步增强其功能。除了基本的阅读功能，它还提供了StarDict字典和维基百科的查词功能、添加在线OPDS书目和RSS源、空中更新软件、FTP客户端和SSH服务器等高级特性。

开发和支持由全球志愿者社区驱动，用户可以通过报告和修复bug、实现新功能、翻译、完善文档或在论坛上提供帮助来贡献力量。项目接受LiberaPay捐赠以支持持续发展。

## jj-vcs / jj

一个简单而又强大的Git兼容VCS

**作者:** [jj-vcs](https://github.com/jj-vcs) | **仓库:** [Github 链接](https://github.com/jj-vcs/jj) | **语言:** Rust

### 项目简介

jj：一个简单而强大的兼容 Git 的版本控制系统

jj (Jujutsu) 是一个新颖的版本控制系统，设计目标是提供简单易用且功能强大的用户体验，同时兼容现有的 Git 生态系统。它通过将用户界面和核心算法从底层存储系统中抽象出来，实现了以 Git 仓库作为其物理存储后端，这意味着用户可以继续使用许多熟悉的 Git 工具和平台（如 GitHub）进行协作，同时享受 jj 带来的独特优势。

jj 的核心创新之一是**将工作副本视为一个普通的提交**。文件的修改会自动记录为一个提交，并在后续更改时进行修改。这种“快照”设计简化了用户模型，消除了 Git 中暂存区（index）和 stash 等概念的需要。此外，jj 会记录所有操作，形成一个操作日志，用户可以轻松地撤销任何操作，提高了追溯和纠错的能力。

jj 在处理冲突方面也独具特色，它将**冲突视为一等公民**，记录在提交中。这使得合并操作无论是否有冲突都能成功，用户可以稍后解决冲突，并且冲突的解决方案会被自动传递给后代提交。在功能上，jj Borrow 了 Mercurial 和 Sapling 中的一些概念，例如 revset 语言用于选择提交，并且没有显式的暂存区或命名分支的概念，简化了分支管理。它还提供了丰富的历史重写工具，如 `jj describe` 修改提交描述、`jj diffedit` 编辑提交中的更改、`jj split` 分割提交以及 `jj squash -i` 合并更改到现有提交等。

jj 正在积极开发中，虽然已经相当成熟并被核心开发者广泛使用，但仍有一些实验性功能仍在完善，例如安全、并发的复制能力，允许在分布式文件系统等环境中使用而不会导致仓库损坏。项目欢迎社区贡献，文档齐全，提供了安装指南、教程和与 Git 的对比，帮助用户快速上手。目前，jj 作为 Google 工程师的个人项目起源，虽然有多名 Google 员工参与，但这并非 Google 官方产品。它采用 Apache 2.0 开源许可。

## Mail-0 / Zero

0 – 首款将隐私和安全放在首位的开源电子邮件应用，体验由你掌控的电子邮件。

**作者:** [Mail-0](https://github.com/Mail-0) | **仓库:** [Github 链接](https://github.com/Mail-0/Zero) | **语言:** TypeScript

### 项目简介

Zero：注重隐私安全的开源AI邮件客户端

“Zero”是一款开源的AI邮件解决方案，目标是重塑并改进电子邮件体验，尤其强调用户隐私和数据安全。与许多封闭源代码、收集用户数据或难以自行部署的现有邮件服务不同，Zero提供了高度的透明度、隐私保护和部署灵活性。

Zero的核心特点包括：完全开源，代码透明无隐藏；由AI驱动，通过智能代理和大型语言模型（LLM）增强邮件功能（例如，最近的开发更新提到了集成Google AI SDK以改进邮件撰写和风格提取）；数据隐私优先，承诺不跟踪、不收集、不销售用户数据，同时提醒用户与外部服务（如Gmail）集成时，相关数据受这些服务的隐私政策约束；支持用户轻松自部署，实现强大的自主控制；提供统一收件箱，可连接Gmail、Outlook等多种邮件服务；UI和功能高度可定制，用户可以按照自己的喜好配置邮件体验；以及对开发者友好，具有良好的扩展性和集成能力。

项目的技术栈基于现代可靠技术，包括前端使用Next.js、React、TypeScript、TailwindCSS和Shadcn UI，后端使用Node.js和Drizzle ORM，数据库使用PostgreSQL，认证采用Better Auth和Google OAuth。这为项目的开发和维护提供了坚实的基础。

对于想要试用或开发Zero的用户，项目提供了详细的“开始指南”，包括所需的Node.js、Bun和Docker版本，以及标准和开发者容器两种设置方式。指南详细介绍了代码获取、依赖安装、环境配置（特别是OAuth设置和数据库连接）的步骤，以及数据库常用命令（如创建表、生成/应用迁移、查看数据）等必要信息。

Zero项目积极欢迎社区贡献，提供了贡献指南和翻译指南，鼓励开发者和用户参与到项目的改进中来。该项目已获得许多社区的关注和贡献，并在持续发展中，最新的提交历史显示了活跃的代码更新和功能改进，例如最近的更新涉及迁移到新的技术栈、改进CI工作流、增强邮箱助理功能、添加多语言支持等。项目的开发由一个积极的维护者和贡献者社群共同推进。

总而言之，Zero提供了一个以隐私和安全为核心、融合AI技术、高度可定制且支持自托管的开源邮件客户端，旨在为用户带来现代化的、符合个人需求的电子邮件使用体验。

## alibaba / MNN

MNN：阿里经商业级业务深度检验的超快轻量级深度学习框架。实现全模态大语言模型Android App：[MNN-LLM-Android](./apps/Android/MnnLlmChat/README.md)

**作者:** [alibaba](https://github.com/alibaba) | **仓库:** [Github 链接](https://github.com/alibaba/MNN) | **语言:** C++

### 项目简介

MNN：阿里巴巴的轻量级高性能深度学习框架

MNN 是由阿里巴巴开发的轻量级、高性能深度学习框架，专为设备端推理和训练而设计。它在阿里巴巴内部有丰富的关键业务场景应用经验，例如淘宝、天猫、优酷等30多个App，覆盖直播、短视频、搜索推荐、图像搜索等70多个场景。MNN 也被广泛应用于物联网等嵌入式设备。

MNN 的核心特点包括：

- **轻量级：** 针对设备端优化，无依赖，易于部署到移动设备和嵌入式设备。提供精简构建选项和FP16/Int8量化支持，大幅减少包大小和模型大小。
- **多功能性：** 支持 TensorFlow, Caffe, ONNX, Torchscripts 等多种主流深度学习框架，兼容各种常用的神经网络结构（CNN, RNN, GAN, Transformer）和模型特性（多输入/输出、动态输入、控制流）。Converter 工具支持多种框架的OP转换。
- **高性能：** 在ARM/x64 CPU上通过汇编优化、利用ARMv8.2的FP16支持和VNNI特性，以及 Metal/OpenCL/Vulkan/CUDA 等GPU后端，实现了行业领先的推理性能。采用优化的卷积算法提升效率和稳定性。
- **易用性：** 提供类似 numpy 的数値计算能力，轻量级图像处理模块（MNN-CV），支持模型构建和训练，以及 Python API，方便开发者使用。

MNN 基于 MNN 引擎构建了 **MNN-LLM**（大语言模型运行时方案）和 **MNN-Diffusion**（Stable Diffusion模型运行时方案），旨在将 LLM 和 Stable Diffusion 模型部署到移动手机、PC 和 IOT 等个人平台。MNN-LLM 已支持 Qianwen, Baichuan, Zhipu, LLAMA 等流行 LLM 模型，并提供了 Android 和 iOS 的多模态 LLM App 示例，支持文本到文本、图像到文本、音频到文本以及文本到图片生成等功能。

MNN 是阿里巴巴设备云协同机器学习生产系统 Walle 的基础模块，该系统在 OSDI'22 顶会发表了相关论文，详细阐述了 MNN 的关键设计原则和benchmark测试结果。社区讨论主要集中在钉钉群，欢迎英文用户参与交流。

项目遵循 Apache 2.0 开源许可证，并致謝了多个参考项目。

## pydantic / pydantic

使用 Python 类型提示进行数据验证

**作者:** [pydantic](https://github.com/pydantic) | **仓库:** [Github 链接](https://github.com/pydantic/pydantic) | **语言:** Python

### 项目简介

标题：使用 Python 类型提示进行数据验证的 Pydantic 项目概览

Pydantic 是一个流行的 Python 库，主要用于利用 Python 的类型提示功能进行数据验证和设置管理。该项目提供了快速且可扩展的数据处理能力，能与代码检查工具（linter）和集成开发环境（IDE）良好协作，帮助开发者以清晰、标准化的方式定义数据结构并确保数据的有效性。

核心功能允许用户定义`BaseModel`类，并通过 Python 的类型提示指定字段的类型。Pydantic 会自动验证输入数据是否符合这些类型定义，并在数据不匹配时抛出错误，或者尝试将数据“强制转换”为正确的类型。例如，一个包含字符串 '123' 的 `id` 字段在被定义为 `int` 类型后，Pydantic 会尝试将其转换为整数 123。代码示例清晰地展示了如何定义一个 `User` 模型，包含 `id`、`name`、`signup_ts` 和 `friends` 等字段，以及如何使用外部数据创建并验证一个 `User` 实例。

Pydantic 项目目前活跃开发，最新版本为 V2。V2 相较于 V1.10 是一个重写版本，带来了显著的新功能、性能提升以及一些API上的变化。为了方便用户从 V1 迁移，V2 版本自带了 V1 的最新版本，可以通过 `from pydantic import v1 as pydantic_v1` 导入，从而实现代码库的增量升级。项目的文档详尽，提供了安装指南、使用示例和贡献方式等信息。

总体而言，Pydantic 是 Python 生态系统中一个强大的数据验证工具，尤其适用于需要处理结构化数据（如 JSON、数据库记录等）的应用场景，通过结合 Python 的类型提示，提升了代码的可读性、可靠性和开发效率。项目拥有庞大的用户基础和活跃的贡献者社区，反映了其在业界的重要性和受欢迎程度。
