## 진행 배경
### 생활속 토론의 부재
- 일상 생활에서 토론을 접할 기회가 많지 않습니다.
- 특히 한국 교육 시스템상 토론 수업 진행이 어렵습니다.

### 인터넷 환경에서의 토론
- 누구나 자유롭게 토론할 수 있지만, 사회자가 없어서 절차가 잘 지켜지지 않습니다.

## Gemini API를 활용하는 이유
### 토론의 승/패 결정
- 게임 방식으로 진행하기 떄문에 승/패가 결정되어야 합니다.
- Gemini가 대화 내용을 입력받고 점수를 부여하여 출력합니다.
- 이는 헌재 대규모 언어 모델만이 할 수 있는 역할이라고 생각했습니다.

## 필요한 기능

#### 메인 화면
- 채팅방 생성, 참여 중에 선택

#### 채팅방 생성 화면
- 알파벳, 숫자로 구성된 6자리 랜덤 문자열(초대 코드) 생성
- 초대 코드를 이름으로 방 생성(supabase API 사용)

#### 채팅방 참여 화면
- 초대 코드를 입력하여 해당 채팅방에 접속

#### 주제 선정 & 입장 부여 화면
- 토론 주제 목록 중에서 랜덤으로 선택
- 각 참여자에게 찬성 또는 반대 입장 랜덤 부여
- 카운트가 끝난 후 입론 시작

#### 양측 입론 화면
- 양측이 동시에 입론을 작성
- 타이머를 이용하여 5분의 제한 시간 부여

#### 양측 반론 화면
- 양측이 번갈아 반론을 작성
- 타이머를 이용하여 3분의 제한 시간 부여

#### 양측 최종변론 화면
- 양측이 번갈아 최종변론을 작성
- 타이머를 이용하여 3분의 제한 시간 부여

#### 최종 평가
- 모든 대화 내용을 하나의 문자열로 정리
- Gemini에게 대화 내용을 가지고 점수 계산 및 평가 요청
- 화면에 결과 출력
